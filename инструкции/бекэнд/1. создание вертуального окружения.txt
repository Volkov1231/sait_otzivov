Вертуальное окружение - это изолированная среда, не зависящая от системы. Запуск в нём равносилен запуску проекта на новом компьютере.

СОЗДАНИЕ
Создаётся командами в терминале, внутнри активной папки (которая пишетс\я желтым цветом, переходить по папкам нужно командой cd)

При создании обращаемся к нужной версии питона (на которой хотим запустить проект). Например, python3.9
Если написать просто python, будет выбрана стандартная версия

дальше нужно написать
-m venv venv
Первое venv - это команда для создания виртуального окружения (вирт. окружения)
Второе venv - имя папки. ОНо может быть любым, но рекомендуется использовать venv

в итоге полдучается такая строчка
python -m venv venv



АКТИВАЦИЯ
Была создана папака venv
Внутри неё есть папка Scripts
Внутри неё есть файл Activate
Чтобы активировать нужно написать в терминале команду source и обратиться к этому файлу

В итоге получается строка
source venv/Scripts/Activate

подтверждением тому, что венв активирован служит напдпись venv в терминале до основной строки.

ДЕАКТИВАЦИЯ
Чтобы выйти из решима в.окр. нужно написать команду deactivate



Чтобы VSCode брал за основу эту папку в.окр. надо:
- Открыть любой файл .py
- Внизу справа нажать  на версию питона
- Выбрать путь к venv, если он там есть
- Если его нет, "Введите путь к интерпретатору" и выбрать папку venv, внутри неё Scripts, выбрать файл python.exe
